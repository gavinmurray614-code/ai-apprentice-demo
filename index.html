Technical Implementation Guide for EYRAC Platform

To integrate this surgical input control into the existing learning platform, follow these three architectural steps:

1. Leverage the beforeinput Event

Unlike the standard paste event, beforeinput allows you to intercept the text before it hits the DOM. You can specifically target insertFromPaste and insertFromDrop without breaking standard typing.

2. Implement "Signature" Heuristics

Most manual browser-to-browser copying includes text/html metadata. Most dictation and accessibility tools only provide text/plain. By inspecting the clipboard types during the paste event, you can differentiate between a student copying a solution and a student using a dictation tool.

3. Protect Accessibility with isComposing

Always verify the isComposing property. If true, the input is likely coming from a voice-to-text tool or an IME. These should be white-listed to maintain platform accessibility.

// Surgical Integration Snippet
element.addEventListener('beforeinput', (e) => {
  const isPaste = e.inputType === 'insertFromPaste';
  const hasRichMetadata = lastClipboardTypes.includes('text/html');
  
  if (isPaste && hasRichMetadata && !e.isComposing) {
    e.preventDefault(); // Block manual paste only
    notifyUser("Please type your response manually.");
  }
});
